\textbf{Цель лабораторной работы}: реализовать алгоритмы сжатия по Хаффману.

Для достижения поставленной цели требуется решить следующие \textbf{задачи}.
\begin{enumerate}
\item Описать сжатие по Хаффману.
\item Реализовать сжатие файла.
\item Реализовать возможность считывать данные из файла.
\item Реализовать возможность записать ответ в бинарный файл.
\item Выполнить тестирование реализации разработанного алгоритма.
\end{enumerate}

Согласно варианту, требуется разработать сжатие файла по Хаффману для исходного кода программы на LISP.

   \section{Аналитическая часть} 
\end{center}

\textbf{Сжатие файла по алгоритму Хаффмана}~\cite{book} --- это метод кодирования данных, который использует переменные длины кодовых слов для представления символов в исходном тексте. Он основан на идее присвоения более коротких кодов более часто встречающимся символам и более длинных кодов менее часто встречающимся символам.

\textbf{LISP} --- это  функциональный язык программирования, который известен своим использованием списков как основной структуры данных и кода. Файлы LISP могут содержать определения функций, переменных, структур данных, а также вызовы функций, операции и другие выражения на языке LISP.

    \section{Конструкторская часть}

\subsection{Создание дерева Хаффмана}
\begin{enumerate}
    \item Подсчет частот символов.
    \begin{itemize}
        \item[---] Создать пустой словарь (map) ``freq'', где будут храниться частоты символов.
        \item[---] Пройтись по всем символам в тексте и увеличить частоту символа в словаре ``freq''. 
    \end{itemize}
    \item Создание приоритетной очереди.
        \begin{itemize}
        \item[---] Приоритетная очередь ``pq'' используется для хранения узлов дерева.
        \item[---] Для каждой буквы из словаря ``freq'' создать узел и поместить его в приоритетную очередь ``pq''. 
    \end{itemize}
    \item Построение дерева Хаффмана.
        \begin{itemize}
        \item[---] Выполнять нижеперечисленные действия, пока в приоритетной очереди не останется только один узел.
        \item[---] Извлечь два узла с минимальными частотами (две самые редкие буквы). Создать новый узел с суммой частот этих двух узлов и добавить обратно в очередь.
    \end{itemize}
    \item Получение корня дерева.
    \begin{itemize}
        \item[---] Корень дерева Хаффмана остается в приоритетной очереди после всех итераций. Извлечь его из очереди и присвоить его корню дерева Хаффмана.
    \end{itemize}
\end{enumerate}

\subsection{Кодирование элементов}
\begin{enumerate}
    \item Функция ``encode'' заполняет словарь ``huffmanCode'' кодами Хаффмана для каждой буквы.
    \item Параметры функции.
    \begin{itemize}
        \item[---] ``Node* root'' --- указатель на текущий узел дерева Хаффмана, начиная с корня.
        \item[---] ``string str'' --- строка, содержащая текущий код Хаффмана для данного узла.
        \item[---] ``map<char, string> \&huffmanCode''' --- словарь, в который мы записываем коды Хаффмана для символов. 
    \end{itemize}
    \item Проверка наличия узла.
    \begin{itemize}
        \item[---] Если текущий узел ``root'' равен ``nullptr'', то выйти из функции, так как достигнут конец ветви дерева.
    \end{itemize}
    \item Проверка листовых узлов.
    \begin{itemize}
        \item[---] Провеить, является ли текущий узел листовым, то есть у него нет потомков. Если это лист, записать текущий код ``str'' в словарь ``huffmanCode'' по ключу ``root->ch''. 
    \end{itemize}
    \item Рекурсивный обход дерева.
    \begin{itemize}
        \item[---] Рекурсивно вызвать ``encode'' для левого потомка текущего узла, добавить к текущему коду символа "$0$".
        \item[---] Рекурсивно вызвать ``encode'' для правого потомка текущего узла, добавить к текущему коду символа "$1$".
    \end{itemize}
\end{enumerate}

\subsection{Вычисление коэффициента сжатия}
\begin{enumerate}
    \item Подготовка переменных.
    \begin{itemize}
        \item[---] ``int bit\_1 = text.size() * 8'' рассчитать общее количество битов в исходном тексте. Умножение на 8 осуществляется, потому что каждый символ представляется 8 битами в стандартной кодировке ASCII.
        \item[---] ``int bit\_2 = 0'' эта переменная будет использоваться для подсчета общего количества битов в закодированном тексте.
    \end{itemize}
    \item Подсчет битов закодированного текста.
    \begin{itemize}
        \item[---] Цикл ``for'' проходит по каждой паре в словаре ``huffmanCode'', который содержит символы и их соответствующие коды Хаффмана.
        \item[---] Для каждой пары вычислить количество битов, занимаемых кодом Хаффмана для символа ``pair.second.size()'' и умножить на частоту символа в исходном тексте ``freq.at(pair.first)''. Это происходит потому, что сжатие зависит от того, сколько битов занимает каждый символ после кодирования и как часто он встречается в исходном тексте. Полученные значения присвоить переменной ``bit\_2''.
    \end{itemize}
    \item Вычисление коэффициента сжатия.
    \begin{itemize}
        \item[---] ``(bit\_1 + 0.0) / bit\_2'' Рассчитывает отношение количества битов исходного текста к количеству битов закодированного текста. Используется тип double, чтобы сохранить десятичную часть коэффициента.
    \end{itemize}
\end{enumerate}

Была подключена библиотека <map> --- контейнер, представляющий отображение ключей на значения. Библиотека <vector> --- это  динамический массив, который обеспечивает быстрый доступ к элементам и динамическое изменение размера.Также была подключена библиотека <string> для пользования строками. Был также написан стек на ссылках. Для чтения и записи данных из/в файл была подключена библиотека <fstream>.

\begin{lstlisting}[caption={Исходный код программы для сжатия по Хаффману},label={lst:label1}]
#include <iostream>
#include <fstream>
#include <queue>
#include <map>
#include <vector>
#include <string>

using namespace std;

struct Node {
	char ch;
	int freq;
	Node *left, *right;
};

struct Compare {
	bool operator()(Node* l, Node* r) {
		return (l->freq > r->freq);
	}
};

Node* getNode(char ch, int freq, Node* left, Node* right) {
	Node* node = new Node();
	node->ch = ch;
	node->freq = freq;
	node->left = left;
	node->right = right;
	return node;
}

void encode(Node* root, string str, map<char, string> &huffmanCode) {
	if (root == nullptr) return;
	if (!root->left && !root->right) {
		huffmanCode[root->ch] = str;
	}
	encode(root->left, str + "0", huffmanCode);
	encode(root->right, str + "1", huffmanCode);
}

void writeHuffmanCodesToFile(const map<char, string>& huffmanCode, const string& encodedText, const map<char, int>& freq) {
	ofstream outputFile("C:\\Users\\Kononova\\Desktop\\huffman_codes.bin", ios::binary);

	if (outputFile.is_open()) {
		for (const auto& pair : huffmanCode) {
			outputFile.put(pair.first);

			int frequency = freq.at(pair.first);
			outputFile.write(reinterpret_cast<const char*>(&frequency), sizeof(frequency));

			char codeLength = static_cast<char>(pair.second.length());
			outputFile.put(codeLength);
		}

		for (char ch : encodedText) {
			outputFile.put(ch);
		}

		outputFile.close();
		cout << "Данные успешно записаны в бинарный файл 'huffman_codes.bin'" << endl;
	}
	else {
		cout << "Невозможно открыть файл для записи" << endl;
	}
}

void winnings(string text, const map<char, string>& huffmanCode, const map<char, int>& freq) {
	int bit_1 = text.size() * 8;
	int bit_2 = 0;
	
	for (auto &pair : huffmanCode) {
		bit_2 += pair.second.size() *  freq.at(pair.first);
	}
	double win = (bit_1 + 0.0) / bit_2;
	cout << "Сжатие в " << win << " раз" << endl;
}

void buildHuffmanTree(string text) {
	map<char, int> freq;
	for (char ch : text) {
		freq[ch]++;
	}

	priority_queue<Node*, vector<Node*>, Compare> pq;	

	for (auto pair : freq) {
		pq.push(getNode(pair.first, pair.second, nullptr, nullptr));
	}

	while (pq.size() != 1) {
		Node* left = pq.top(); pq.pop();
		Node* right = pq.top(); pq.pop();

		int sumFreq = left->freq + right->freq;
		pq.push(getNode('\0', sumFreq, left, right));
	}

	Node* root = pq.top();

	map<char, string> huffmanCode;
	encode(root, "", huffmanCode);

	string encodedText;
	for (char ch : text) {
		encodedText += huffmanCode[ch];
	}

	cout << "Частоты букв: " << endl;
	for (auto pair : freq) {
		cout << pair.first << " : " << pair.second << endl;
	}
	cout << endl;

	cout << "Код букв: " << endl;
	for (auto pair : huffmanCode) {
		cout << pair.first << " : " << pair.second << endl;
	}
	cout << endl;

	cout << "Закодированный текст по Хаффману: " << encodedText << endl;

	writeHuffmanCodesToFile(huffmanCode, encodedText, freq);
	cout << endl;
	winnings(text, huffmanCode, freq);
}

int main() {
	setlocale(0, "");
	ifstream file("C:\\Users\\Kononova\\Desktop\\huff.lisp");
	string text = "";
	if (file.is_open()) {
		string text_help = "";
		while (getline(file, text_help)){
			text += text_help;
			text_help = "";
		}
		file.close();
	}
	else {
		cout << "Невозможно открыть файл";
		return 0;
	}
	cout << "Прочитанная фраза: " <<text << endl;
	buildHuffmanTree(text);
	
	return 0;
}

Анализируя выполненные тесты, можно отметить, что при увеличении объема данных алгоритм Хаффмана становится более эффективным в сжатии по сравнению с простым методом сжатия. Это происходит благодаря созданию оптимального кода для представления данных: наиболее часто встречающиеся символы кодируются более короткими кодами, что позволяет значительно сократить общий размер данных.

Однако, когда данные становятся более случайными или каждый символ встречается с примерно одинаковой частотой, эффективность сжатия по Хаффману может уменьшаться. Это происходит из-за того, что оптимальное кодирование для равномерно распределенных данных не обеспечивает большого преимущества в сжатии.

Таким образом, при увеличении объема данных сжатие по Хаффману обычно стремится к более эффективному сжатию, но его эффективность может варьироваться в зависимости от характера данных и распределения символов в них.
